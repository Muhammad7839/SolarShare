Methodology

Transparent scoring logic with source-linked market context.

SolarShare converts pricing, reliability, and distance signals into ranked options. Priority mode controls sorting behavior while preserving visible tradeoffs.

Scoring approach

Effective price is computed from base price, time-of-use adjustment, distance sensitivity, and a reliability bonus. Customer-selected priorities then determine final ranking order.

The top recommendation includes an explanation string so users understand why the selected option wins.

Inputs used

  • Location text normalized and geocoded
  • State-level retail utility baseline
  • Shortwave radiation and cloud cover context
  • Provider-specific reliability and distance factors

Priority modes

  • Balanced: overall tradeoff optimization
  • Lowest Cost: monthly cost minimization
  • Highest Reliability: confidence-first ranking
  • Closest Distance: proximity-first ordering

Live Data Integrity

Geocoding

OpenStreetMap/Nominatim for location-to-coordinate resolution.

Weather context

Open-Meteo irradiance and cloud data used in reliability adjustments.

Utility baseline

U.S. EIA retail electricity rates, with latest period metadata when available.

Resilience and Safety

  • If external APIs fail, fallback-safe values are used and clearly labeled in the UI.
  • Response payload includes source URLs and observation timestamps.
  • Contact inquiries are validated server-side and rate-limited.
  • Request IDs are attached to responses for traceability and support triage.
